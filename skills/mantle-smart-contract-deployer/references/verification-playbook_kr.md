# 검증 플레이북

Mantle 익스플로러에서 외부적으로 배포된 컨트랙트 검증을 계획하거나 검증하려면 이 워크플로우를 사용하세요.

## 필수 입력

- 배포된 컨트랙트 주소
- 컨트랙트 소스 경로 및 컨트랙트 이름
- 컴파일러 버전
- 최적화 활성화/실행 횟수
- 생성자 인수 (원시 및 인코딩)
- 링크된 라이브러리 주소 (있는 경우)

## 단계

1. 대상 주소에 배포된 바이트코드가 있는지 확인합니다.
2. 익스플로러 API/UI 제출을 위한 검증 페이로드를 준비합니다 (툴링을 사용할 수 없을 때 수동 제출 지시 제공).
3. 성공/실패/타임아웃까지 익스플로러 응답에서 검증 상태를 폴링합니다.
4. 검증 링크 또는 식별자를 기록합니다.

## 흔한 실패 및 수정

- 컴파일러 불일치:
  - 배포 시 사용된 정확한 컴파일러 버전으로 재빌드합니다.
- 최적화 불일치:
  - 활성화 플래그와 실행 횟수를 정확히 일치시킵니다.
- 생성자 인수 불일치:
  - 정확한 인수 순서 및 유형으로 다시 인코딩합니다.
- 메타데이터/라이브러리 불일치:
  - 완전히 정규화된 라이브러리 매핑을 제공합니다.

## 증거 요구사항

- 전체 요청 페이로드 스냅샷 유지 (비밀 제외).
- 익스플로러 응답 본문과 타임스탬프 유지.
- 에이전트가 배포/검증 트랜잭션을 제출했다고 주장하지 마세요; 실행은 외부에서 발생합니다.
- 명시적 성공 응답 없이 "검증됨"을 주장하지 마세요.
