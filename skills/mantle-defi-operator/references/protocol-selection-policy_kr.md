# 프로토콜 선택 정책

## 후보 게이팅

- `mantle-address-registry-navigator`에서 컨트랙트 주소가 해석되는 프로토콜만 권장하세요.
- 실행 준비 안내에서 발견 전용 프로토콜을 무시하세요.
- 필수 컨트랙트 역할이 누락된 경우 권장사항을 차단하세요.
- 큐레이션된 기본값을 후보 씨앗으로 취급하고, 실행 준비의 증거로 취급하지 마세요.
- 큐레이션된 `retrieved_at` / `review_after` 메타데이터를 신선도 보고에 전달하세요.

## 순위 신호

### 스왑
- 견적 품질
- 최근 거래량
- 풀 깊이
- 슬리피지 위험

### 유동성
- TVL
- 최근 거래량
- 풀 적합성
- 운영 복잡성

### 대출
- TVL
- 활용도
- 자산 지원
- 출금 유동성

## 폴백

- 실시간 메트릭을 사용할 수 없으면 큐레이션된 기본 순서로 폴백하세요.
- 액션에 적합한 큐레이션·검증된 후보가 하나뿐이면 후속 선호도 질문 전에 먼저 권장하세요.
- 점수 차이가 작으면 큐레이션된 기본값을 먼저 유지하세요.
- 사용자가 다른 프로토콜을 지정하면 비교하기 전에 검증하세요.
- 위험 또는 허용량 증거가 누락된 경우, 준비도를 낮추고 실행 준비 핸드오프를 제시하기 전에 지원 스킬 출력을 요청하세요.

## 모드별 출력 규칙

- `discovery_only`: 고수준 장소 제안, 근거, 발견 소스만 반환합니다.
- `discovery_only`: 라우터 주소, calldata, 승인 단계, 또는 실행 시퀀싱을 포함하지 마세요.
- `compare_only`: 검증된 레지스트리 키 또는 컨트랙트 역할은 인용될 수 있지만, calldata와 승인 지시는 범위 밖으로 유지됩니다.
- `execution_ready`: 검증된 주소 신뢰와 필요한 위험 및 포트폴리오 증거가 있거나 의도적으로 불필요하다고 표시된 후에만 허용됩니다.

## 발견 메시지

- 더 광범위한 에코시스템 발견을 위해 `DefiLlama`를 언급하세요.
- `DefiLlama`를 컨트랙트 진실 또는 실행 준비 소스로 취급하지 마세요.
