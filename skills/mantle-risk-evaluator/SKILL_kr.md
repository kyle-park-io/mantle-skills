# Mantle 위험 평가기

## 개요

필수 사전 비행 체크리스트를 실행하고 명확한 `pass`, `warn`, 또는 `block` 판정을 반환합니다. 중요한 조건이 실패할 때 안전하지 않은 실행을 방지합니다.

## 워크플로우

1. 인텐트 입력을 정규화합니다:
   - 작업 유형
   - 토큰 입력/출력 및 금액
   - 대상 프로토콜/라우터/풀 주소
   - 사용자 위험 매개변수 (슬리피지 한도, 데드라인, 최대 가스 선호도)
2. 체크리스트를 실행합니다 (아래 체크리스트 섹션 참조).
3. 임계값 표를 적용합니다 (아래 임계값 섹션 참조). 사용자 지정 매개변수를 먼저 사용하고, 사용자 제약이 없을 때만 기본값을 사용합니다.
4. 각 항목을 분류합니다:
   - `pass`
   - `warn`
   - `fail`
5. 최종 판정을 생성합니다:
   - `pass`: 실패 없음, 선택적 경고.
   - `warn`: 중요 실패 없음, 사용자 확인 필요.
   - `block`: 하나 이상의 중요 실패.

## 필수 입력 필드

다음 항목 모두 인텐트에 있어야 합니다. 누락된 항목이 있으면 즉시 최종 판정을 `block`으로 설정하고 차단 이유를 "필수 필드 누락"으로 표시하세요:

- `operation_type`
- `chain/environment`
- 토큰 입력/출력 및 금액
- 대상 컨트랙트/라우터/풀 주소

## 체크리스트

### 1. 슬리피지 검사
- "제안된 슬리피지"는 이 트랜잭션에 대해 사용자(또는 프로토콜)가 설정한 슬리피지 허용치입니다.
- 사용자가 직접 설정했는지 여부에 관계없이 항상 제안된 슬리피지를 아래 기본 임계값 표와 비교하세요.
- 사용자가 높은 슬리피지 허용치를 선택했다고 해서 안전한 것은 아닙니다 — 기본 임계값이 여전히 적용됩니다.
- 사용자가 별도로 더 엄격한 한도를 지정하면 더 제한적인 것을 적용하세요.
- 제안된 슬리피지가 실패 임계값(기본적으로 > 1.0%)을 초과하면 실패합니다.

### 2. 유동성 깊이 검사
- 견적/시뮬레이션 컨텍스트에서 가격 영향을 추정합니다.
- 중간 영향 시 경고, 심각한 영향 시 실패합니다 (임계값 참조).
- 유동성 데이터를 사용할 수 없으면 "신뢰도 감소" 노트와 함께 warn으로 설정합니다.

### 3. 주소 안전성 검사
- 신뢰할 수 있는 레지스트리/툴링에 대해 모든 주소를 검증합니다.
- `pass`: 신뢰할 수 있고 검증된 소스.
- `warn`: 알 수 없지만 명시적으로 표시되지 않음.
- `fail`: 표시됨, 블랙리스트에 올려짐, 또는 잘못된 형식의 주소.

### 4. 허용량 범위 검사
- 의도된 금액에 필요한 것보다 광범위한 승인을 감지합니다.
- `pass`: 기존 허용량이 의도된 지출 범위에 맞음.
- `warn`: 허용량이 의도된 지출보다 실질적으로 큼.
- `fail`: 명시적 사용자 확인 없이 새 또는 기존 무제한에 가까운 승인. 무제한에 가까운 발견: 원시 허용량 `>= 2^255`.

### 5. 가스 및 데드라인 유효성
- 최근 기준선 대비 가스 추정의 합리성을 확인합니다.
- 트랜잭션 데드라인이 오래되지 않았고 지나치게 길지 않은지 확인합니다.
- 아래 임계값을 적용합니다.

## 임계값 (사용자가 지정하지 않은 경우 기본값)

| 검사 | 경고 | 실패 |
|---|---|---|
| 슬리피지 | > 0.5% | > 1.0% |
| 예상 가격 영향 | > 2% | > 5% |
| 데드라인 범위 | > 20분 | > 60분 |
| 기준선 대비 가스 편차 | > 20% | > 40% |

## 판정 규칙

- 체크리스트 항목에서 `fail` => 최종 판정 **block**.
- `fail` 없고 최소 하나의 `warn` => 최종 판정 **warn**.
- 모든 검사 `pass` => 최종 판정 **pass**.

### 신뢰도 정책
- `high`: 필요한 모든 신호가 있고 일관성이 있음.
- `medium`: 하나의 중요하지 않은 신호 누락.
- `low`: 주요 신호 누락 (예: 유동성 데이터 없음 또는 확인되지 않은 주소 출처).
- 신뢰도가 낮으면 판정을 한 수준 낮춥니다 (`pass` → `warn`, `warn` → `block`).

## 차단 조건

- 계획된 슬리피지가 사용자 한도 또는 기본값 1.0%를 초과합니다.
- 주소 위험 검사 실패 (표시됨, 블랙리스트, 또는 잘못된 형식의 상대방).
- 유동성 깊이가 5%를 초과하는 심각한 가격 영향을 나타냅니다.
- 기준선 대비 가스 편차가 40%를 초과합니다.
- 명시적 사용자 확인 없이 무제한에 가까운 허용량 (>= 2^255).
- 필수 인텐트 필드 누락 (안전하게 평가할 수 없음).

## 출력 형식

항상 이 정확한 구조를 사용하세요. 필드 누락으로 인해 일찍 차단할 때도 모든 응답에 다섯 가지 체크리스트 항목 모두를 포함해야 합니다. 평가할 수 없는 검사(예: 필수 입력이 누락된 경우)는 이유를 설명하는 `fail`로 표시하세요 (예: "평가 불가 -- 필수 필드 누락").

```text
Mantle 사전 비행 위험 보고서 (Mantle Preflight Risk Report)
- operation:
- environment:
- evaluated_at_utc:

체크리스트 (Checklist)
- slippage_check: pass | warn | fail
  details:
- liquidity_depth_check: pass | warn | fail
  details:
- address_safety_check: pass | warn | fail
  details:
- allowance_scope_check: pass | warn | fail
  details:
- gas_and_deadline_check: pass | warn | fail
  details:

최종 판정 (Final Verdict)
- status: pass | warn | block
- blocking_reasons:
- user_action_required:
```
