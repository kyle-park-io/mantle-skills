# 위험 체크리스트

실행 인텐트가 승인되기 전에 모든 검사를 적용하세요.

## 입력 완전성

- `operation_type` 있음
- `chain/environment` 있음
- 토큰 및 금액 필드 있음
- 대상 컨트랙트/라우터/풀 주소 있음

필수 필드가 누락된 경우 실패합니다.

## 슬리피지 검사

- 제안된 슬리피지를 사용자 정의 한도와 비교합니다.
- 사용자 한도가 없으면 `risk-threshold-guidance.md`의 기본값을 적용합니다.
- 한도 초과 시 실패합니다.

## 유동성 깊이 검사

- 견적/시뮬레이션 컨텍스트에서 가격 영향을 추정합니다.
- 중간 영향 시 경고, 심각한 영향 시 실패합니다 (임계값 기준).
- 유동성 데이터를 사용할 수 없으면 신뢰도 감소와 함께 warn으로 설정합니다.

## 주소 안전성 검사

- 신뢰할 수 있는 레지스트리/툴링에 대해 모든 주소를 검증합니다.
- 알 수 없거나 의심스럽거나 블랙리스트된 주소를 표시합니다.
- 블랙리스트/명시적으로 표시된 주소에 대해 실패합니다.

## 허용량 범위 검사

- 의도된 금액에 필요한 것보다 광범위한 승인을 감지합니다.
- 광범위한 허용량에 대해 경고합니다.
- 사용자 확인 없이 새로운 무제한 승인이 필요한 경우 실패합니다.

## 가스 및 데드라인 유효성

- 최근 기준선 대비 가스 추정의 합리성을 확인합니다.
- 트랜잭션 데드라인이 오래되지 않았고 지나치게 길지 않은지 확인합니다.
- 임계값 프로필에 따라 경고 또는 실패합니다.

## 최종 규칙

- 중요 카테고리에서 `fail` => 최종 판정 `block`.
- `fail` 없고 최소 하나의 `warn` => 최종 판정 `warn`.
- 모든 검사 통과 => 최종 판정 `pass`.
