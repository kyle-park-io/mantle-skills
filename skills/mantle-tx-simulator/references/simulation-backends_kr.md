# 시뮬레이션 백엔드

가용성, 충실도, 재현성 요구사항에 따라 백엔드를 선택하세요.
이 백엔드들은 외부에 있으며 CLI 툴체인 외부에서 실행해야 합니다.

## 백엔드 옵션

### 로컬 포크 (Anvil 또는 동급)

- 결정론적 로컬 재생 및 빠른 반복에 가장 적합합니다.
- 아카이브 가능한 RPC와 올바른 포크 블록 설정이 필요합니다.
- 커스텀 호출 시퀀스를 사용한 심층 디버깅에 좋습니다.

### 관리형 시뮬레이션 API (Tenderly 또는 동급)

- 풍부한 추적으로 빠른 호스팅 시뮬레이션에 가장 적합합니다.
- 팀 워크플로우 및 공유 가능한 시뮬레이션 아티팩트에 좋습니다.
- 공급자 가동 시간 및 계정/프로젝트 설정에 의존합니다.

## 선택 휴리스틱

1. 결정론적 재현성 또는 커스텀 사전 상태 설정이 필요할 때 로컬 포크를 사용합니다.
2. 속도, 추적, 협업이 필요할 때 관리형 시뮬레이터를 사용합니다.
3. 하나의 백엔드가 실패하면 결론이 나지 않음을 선언하기 전에 대체 백엔드로 한 번 재시도를 요청합니다.

## 최소 캡처 필드

- 백엔드 이름
- 네트워크/체인 ID
- 시뮬레이션 타임스탬프 (UTC)
- 트랜잭션 페이로드
- 성공/되돌리기 상태
- 가스 추정
- 디코딩된 이벤트/로그 (있는 경우)
- 상태 변경 요약

## 결론 없음 조건

- 추적 없는 백엔드 오류
- 해결되지 않은 calldata 디코딩
- 사용자 대면 요약에 필요한 토큰 메타데이터 누락

이런 결론 없음 조건이 존재할 때 `status: inconclusive` 및 `do_not_execute_reason`을 반환합니다.
