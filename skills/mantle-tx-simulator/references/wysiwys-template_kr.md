# WYSIWYS 템플릿 (서명하면 이렇게 됩니다)

시뮬레이션 출력을 평문 서명 기대사항으로 변환하려면 이 템플릿을 사용하세요.

## 필수 추출

- 사용자 지갑에서 차감된 자산
- 사용자 지갑에 입금된 자산
- 네이티브 토큰의 수수료 추정
- 승인 변경사항 (있는 경우)
- 중요 주의사항 (슬리피지, 데드라인, 해결되지 않은 불확실성)

## 평문 형식

```text
이 트랜잭션을 브로드캐스트하면:
- 지출하게 됩니다: <금액 토큰>
- 최소 받게 됩니다: <금액 토큰>
- 예상 네트워크 수수료: <금액 네이티브 토큰>
- 추가 승인 변경사항: <설명 또는 없음>
```

## 규칙

- 결과가 슬리피지에 의존하는 경우 "최소" 표현을 사용하세요.
- 출력 토큰 금액이 불확실한 경우 명시적으로 그렇게 명시하세요.
- 시뮬레이션이 되돌려진 경우 낙관적인 받는 금액을 제공하지 마세요.
- 시뮬레이션을 보장된 최종 실행으로 절대 제시하지 마세요.

## 되돌리기 문구 템플릿

```text
실행 전 시뮬레이션이 실패했습니다.
- 가능한 실패 지점: <함수/단계>
- 관찰된 오류: <디코딩된 메시지 또는 원시 되돌리기 데이터>
- 조치: 문제가 해결될 때까지 do_not_execute.
```
