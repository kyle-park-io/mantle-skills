# 허용량 위험 규칙

지출자 승인 노출을 분류하려면 이 규칙을 사용하세요.

## 입력

- 토큰 주소
- 지출자 주소
- 허용량 원시 값
- 토큰 소수점 (알려진 경우)
- `mantle-cli account allowances` 출력의 `is_unlimited` 플래그 (있으면 선호)

## 위험 수준

- `low`
  - 허용량이 0임.
  - 또는 허용량이 엄격하게 제한되어 있고 지갑 잔액/예상 사용량보다 명확히 낮음.
- `medium`
  - 허용량이 0이 아니고 즉각적인 예상 사용량보다 크지만 여전히 제한됨.
- `high`
  - 허용량이 예상 사용량에 비해 매우 큼.
  - 또는 허용량이 의도적으로 광범위하고 사용자 의도가 불분명함.
- `critical`
  - 도구 출력의 `is_unlimited=true`.
  - 또는 허용량이 최대 정수 승인과 같거나 실질적으로 근접함.
  - 일반적인 근최대 감지: 값 >= `2^255`.

## 보고 규칙

- 항상 각 위험 레이블에 근거 텍스트를 포함하세요.
- `mantle://registry/protocols`에서 확인되거나 사용자가 확인한 프로토콜이 아니면 지출자 신뢰 상태를 `unknown`으로 표시하세요.
- 요약 상단에 모든 `high` 및 `critical` 승인을 강조하세요.
- 토큰 소수점이 누락된 경우 원시 값을 사용하여 분류하고 신뢰도를 낮추세요.

## 주의사항

- 무제한 승인이 자동으로 악의적인 것은 아닙니다; 이것은 노출 신호입니다.
- 위험 점수는 휴리스틱이며 프로토콜 신뢰 검사 및 인텐트 컨텍스트와 결합해야 합니다.
